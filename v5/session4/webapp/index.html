W3 — Dataset Selection

Selected Datasets

Two complementary datasets selected to demonstrate the full pipeline and the Sovereign Indic Exception. The lordx64/reasoning dataset is explicitly excluded.

⚠
Why These Three Datasets?
Wikipedia provides clean, well-studied English prose with known noise patterns: disambiguation pages, stub articles, heavy citation markup, and math-symbol-heavy content. It exercises every heuristic rule in Stage 4.

Sangraha (Indic) demonstrates the Sovereign Indic Exception — where English Gopher thresholds fail on Telugu's agglutinative morphology, where ZWNJ/ZWJ must be preserved for Devanagari ligatures, and where AADHAAR/PAN regex patterns become relevant for PII scrubbing.

C4 en.noclean (Web Crawl) is unfiltered CommonCrawl — the same data the trainer benchmarks apply to. Stage 3 (Lang ID) and Stage 4 (Quality) do the heavy lifting. This is why pre-curated sources like Wikipedia survive far more than raw web data.

W7 — Manifest Explorer

Provenance & Manifest Schema

Every admitted document receives a deterministic manifest. The shard_id is derived from content SHA-256, ensuring reproducibility across pipeline runs.

📌 Sample Manifest (Wikipedia shard)

🔒 Pipeline Improvements in Manifest Stage

  • ✅ Deterministic shard_id: shard_{sha256[:12]} — content-addressed, reproducible across runs
  • ✅ Real token counts: BPE-aware subword counting replaces approximation (10× undercount for Indic)
  • ✅ Script hash: Records cleaning_script_hash so pipeline changes are detectable
  • ✅ ISO-8601 timestamp: Full provenance trail per ingest run
  • ✅ License gate: Unknown or unsafe licenses blocked automatically
  • ✅ lang_distribution: Per-shard language breakdown (bilingual shards supported)
